用PyQt6制作《冰与火之舞》游戏复刻。

/audio中存放了关卡音乐文件，/levels中存放了关卡数据，其中记录了关卡的BPM(即小球每分钟转过多少次180°)，每个节拍的角度。

游戏界面由地图和两个小球组成，记为小球A和小球B。

地图由若干个节点和它们之间的连线组成，所有连线长度相等。

在开始游戏前的3拍时，小球A在第0号节点固定，小球B会以开始游戏后的旋转速度的2倍从第1号节点处出发开始旋转3整圈，第3次落回1号节点时游戏正式开始。每当前面的小球落在n号节点上后就会固定住，改为后面的小球旋转，直到从n-1号节点运动到n+1号节点。连线(n-1,n),(n,n+1)之间旋转的角度就对应关卡文件中一个节拍的角度。特别地，如果n号节点处的事件为reverse，连线会反转旋转方向(顺时针->逆时针或者反之)。

开始游戏后小球落在节点上时，玩家需要按下键盘上除了Esc以外的任意按键来踩上节奏。玩家成功按下的条件是小球此时多转或者少转的角度在30°以内。如果玩家没有按下或者差的太多，游戏立即失败，按除了Esc以外的任意键重新开始。玩家成功按下后，即使前面的小球和节点位置有偏移，即少转/多转x°，也会立即强制修正到节点的正确位置，并让后面的小球瞬移到少转/多转x°的位置来确保下一次玩家操作得当能够正确踩点。

如果成功按下节点事件为complete，游戏胜利。
