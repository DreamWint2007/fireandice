# FireAndIce 功能文档

## 1. 项目概述

本项目包含两个核心模块：

- `main.py`：PyQt6 实现的《冰与火之舞》玩法复刻（关卡播放与判定）。
- `level_generator.py`：PyQt6 + librosa 实现的关卡生成器（从音频提取节拍并输出关卡 JSON）。

资源目录约定：

- `audio/`：音频文件（如 `test1X.mp3`）。
- `levels/`：关卡文件（如 `test1X.json`）。

---

## 2. 游戏主程序（`main.py`）

## 2.1 启动方式

- 默认启动（自动选择 `levels/test1X.json` 或 `levels` 首个 JSON）：
  - `python main.py`
- 指定关卡：
  - `python main.py levels/test1X.json`
- 指定模式：
  - `python main.py levels/test1X.json default`
  - `python main.py levels/test1X.json auto`
  - `python main.py levels/test1X.json hard`

## 2.2 模式说明

- `default`：默认判定模式。
- `auto`：自动展示关卡，不需要按键。
- `hard`：更严格失败阈值（见判定规则）。

## 2.3 玩法与流程

- 开场有 3 拍预备阶段：
  - 小球 B 以正式速度 2 倍旋转 3 圈后进入正式开始。
- 正式阶段两球交替作为“固定球/运动球”，按关卡角度旋转。
- 若节拍事件是 `reverse`，旋转方向会翻转。
- 节拍事件是 `complete` 时判定胜利。

## 2.4 判定规则

- 成功窗口：偏差在 `30°` 以内。
- 失败阈值：
  - `default`：`60°`
  - `hard`：`30°`
- 成功踩点后会执行位置修正，并将偏移传递到下一拍（连贯判定体验）。

## 2.5 键位说明

- `Esc`：退出游戏。
- `M`：循环切换模式（`default -> auto -> hard -> default`）。
- 其他任意键：
  - 进行中用于踩点（非 `auto` 模式）。
  - 失败或胜利后用于重开。

## 2.6 视图与镜头

- 地图采用“局部视图 + 相机跟随”显示，不是整图一次性铺满。
- 相机以平滑方式移动，减少每拍抖动。

## 2.7 音乐播放

- 会尝试播放与关卡同名的 MP3（`audio/<关卡名>.mp3`）。
- 音乐在正式开始时播放。
- 失败、胜利、重开会停止当前音乐，避免叠播。

## 3. 关卡生成器（`level_generator.py`）

## 3.1 功能列表

- 选择音频文件（默认从 `audio/` 打开）。
- 设置 `reverse` 事件比例（默认 0%）。
- 设置 BPM 猜测起点（默认 `150 BPM`）。
- 生成关卡 JSON 到 `levels/`。
- 一键测试（自动调用 `main.py` 并以 `auto` 模式运行）。

## 3.2 节拍分析流程

- 使用 `librosa` 分析音频 onset 强度。
- 通过 `beat_track(start_bpm=用户设置值)` 估算 BPM。
- 使用 onset 检测得到时间点，做噪声过滤与节拍间隔分析。
- 以当前最小角度规则进行量化，得到关卡角度序列。

## 3.3 当前角度策略

- 当前最小角度步进为 `90°`（`ANGLE_QUANTUM = 90.0`）。
- 量化策略偏向 `180°` 主节拍：
  - 仅在明显接近半拍时保留 `90°`。
  - 连续 `90° + 90°` 会合并为 `180°`。

## 3.4 输出规则

- `Beats` 默认事件为 `none`。
- 按设定概率随机插入 `reverse`。
- 最后一拍事件强制写为 `complete`。

---

## 4. 关卡文件格式（JSON）

示例结构：

```json
{
  "levelname": "test1X",
  "initial_bpm": 152.0,
  "Beats": [
    { "angle": 180.0, "event": "none" },
    { "angle": 90.0, "event": "reverse" },
    { "angle": 180.0, "event": "complete" }
  ]
}
```

字段说明：

- `levelname`：关卡名称（通常取音频文件名）。
- `initial_bpm`：初始 BPM（本项目语义下，一拍对应 180°）。
- `Beats`：节拍数组，每项包含：
  - `angle`：该拍旋转角度。
  - `event`：`none` / `reverse` / `complete`。
